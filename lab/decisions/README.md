# Decisions

Decision notes capture stable conclusions that contributors should treat as part of the project's working contract.

## Conventions

- Use zero-padded numeric prefixes such as `0001-...`.
- Keep decision notes short.
- Focus on the decision, why it was made, and what it changes.
- If a decision is replaced, mark it `superseded` and link the replacement.
